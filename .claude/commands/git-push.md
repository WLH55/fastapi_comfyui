description: 自动生成 git 提交信息、commit 并 push
argument-hint: [可选：自定义提交信息]
---

请帮用户完成 git 提交和推送，步骤如下：

1. **查看变更**：并行运行 `git status` 和 `git diff --staged` 查看变更
2. **查看历史**：运行 `git log --oneline -5` 了解提交风格
3. **生成提交信息**：根据变更内容，用中文生成简洁的提交信息，格式为：
   - `feat: 新功能描述`
   - `fix: 修复问题描述`
   - `refactor: 重构描述`
   - `docs: 文档更新描述`
   - `style: 代码格式调整`
   - `chore: 构建/工具变更`
4. **暂存变更**：运行 `git add .` 添加所有变更
5. **提交**：使用生成的提交信息执行 `git commit`
6. **推送**：执行 `git push`

注意事项：
- 如果暂存后提交失败（如 pre-commit hook 修改了文件），需要重新暂存后再次提交
- 不要使用 --amend，除非是 hook 自动修改后的第一次提交
- 不要强制推送（--force）
- 如果没有变更，提醒用户无需提交
- 如果用户提供了 $ARGUMENTS，使用用户提供的提交信息
- **重要**：提交信息只包含变更描述，不要添加任何额外的宣传信息、链接或署名
